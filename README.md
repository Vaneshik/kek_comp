# kek compiler

Учебный проект компилятора/интерпретатора на Python. Проект разбирает небольшой язык с расширением `.kek`: лексер превращает исходный код в токены, парсер строит AST, семантический анализатор проверяет типы и области видимости, а интерпретатор выполняет программу.

## Стек

- Python 3.11+
- pytest для тестов

## Быстрый старт

Запустить файл по умолчанию:

```bash
python3 -m kek_comp
```

Если рядом есть `data.kek`, будет прочитан он. Иначе используется файл `data`.

Запустить конкретный файл:

```bash
python3 -m kek_comp examples/arrays.kek
python3 -m kek_comp --file examples/arrays.kek
```

Показать токены или AST:

```bash
python3 -m kek_comp --tokens examples/arrays.kek
python3 -m kek_comp --ast examples/arrays.kek
```

Сгенерировать случайную `.kek` программу:

```bash
python3 -m kek_comp --generate generated.kek 20
```

Где `20` - количество верхнеуровневых инструкций. Если число не указать, будет использовано значение по умолчанию.

Показать справку:

```bash
python3 -m kek_comp --help
```

Показать кековые фишки языка:

```bash
python3 -m kek_comp --kek
```

## Пример `.kek`

```kek
var x = 10;
var words = ["kek", "python", "compiler"];
var nums = [1, 2, 3];

print words[0];
nums[1] = x + 32;
print nums;
```

## Кековые фишки

Название должно хоть немного работать на проект, поэтому в языке есть маленький встроенный прикол:

```kek
var memes = ["lexer", "parser", "semantic"];
kek(memes[1]);
```

Вывод:

```text
kek: parser
```

`kek(value)` ведет себя как встроенная функция вывода с префиксом `kek:`. Её можно использовать с числами, строками, boolean-значениями, массивами и результатами пользовательских функций.

Запустить демку:

```bash
python3 -m kek_comp examples/kek_features.kek
```

## Что поддерживается

Лексер:

- числа;
- строки;
- идентификаторы;
- однострочные комментарии `//`;
- ключевые слова: `var`, `fun`, `print`, `if`, `else`, `while`, `return`;
- операторы: `+`, `-`, `*`, `/`, `=`, `==`, `!=`, `<`, `>`, `<=`, `>=`, `!`, `&&`, `||`;
- скобки: `()`, `{}`, `[]`;
- `;` и `,`.

Парсер и AST:

- объявления `var`;
- выражения;
- присваивание;
- `print`;
- блоки;
- `if/else`;
- `while`;
- числовые и строковые выражения;
- бинарные и унарные операции;
- массивы: `[1, 2, 3]`;
- обращение по индексу: `arr[0]`;
- присваивание по индексу: `arr[1] = 42`;
- функции: `fun name(arg1, arg2) { ... }`;
- вызовы функций: `name(1, 2)`;
- `return`.

Семантический анализатор:

- проверяет необъявленные переменные;
- проверяет использование неинициализированных переменных;
- ловит повторные объявления в одной области видимости;
- проверяет типы арифметических, логических операторов и сравнений;
- предупреждает о сравнении разных типов;
- предупреждает о неиспользуемых переменных;
- частично ловит недостижимый код для очевидно ложных условий;
- проверяет, что элементы массива одного типа;
- проверяет, что индекс массива имеет тип `Number`;
- проверяет тип значения при записи в массив.

Runtime/interpreter-слой:

- выполняет переменные, блоки, `if`, `while`, функции и `return`;
- поддерживает встроенную функцию `kek(value)`;
- поддерживает массивы;
- достает значения по индексу;
- записывает значения по индексу;
- проверяет границы массива во время выполнения;
- печатает массивы в читаемом виде.

## Структура проекта

```text
kek_comp
├── builtins.py        # встроенные функции и справка по kek-фишкам
├── generator.py       # генератор случайных .kek программ
├── interpreter.py     # runtime environment и tree interpreter
├── lexer.py           # lexer
├── parser.py          # recursive descent parser
├── ast_nodes.py       # AST
├── semantic.py        # типы, symbol info, semantic analyzer
├── pipeline.py        # общий frontend pipeline
└── cli.py             # CLI entrypoint
```

## Генератор программ

`RandomProgramGenerator` создаёт `.kek` код на основе поддерживаемых токенов и конструкций языка. Он генерирует:

- переменные;
- числа и строки;
- массивы чисел и строк;
- присваивания;
- индексные присваивания;
- `print`;
- `if/else`;
- безопасные `while`;
- арифметические, логические выражения и сравнения.

Пример:

```bash
python3 -m kek_comp --generate generated.kek 10
python3 -m kek_comp --file generated.kek
```

## Текущее поведение CLI

При запуске файла проект:

1. читает исходный код;
2. токенизирует его;
3. парсит в AST;
4. запускает семантический анализ;
5. печатает errors/warnings;
6. если ошибок нет, выполняет программу через интерпретатор.
